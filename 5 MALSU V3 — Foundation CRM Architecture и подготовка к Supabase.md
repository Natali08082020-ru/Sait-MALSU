# MALSU V3 — Foundation, CRM Architecture и подготовка к Supabase

## Главная задача

Подготовить проект MalSu к переходу на полноценную CRM и Supabase.

На данном этапе НЕ подключать Supabase.

Не создавать серверную часть.

Не внедрять авторизацию.

Не менять внешний вид сайта без необходимости.

Цель этапа:

Создать архитектуру, которая позволит подключить Supabase и админ-панель с минимальными изменениями кода.



# ЭТАП 1. Очистка и подготовка проекта

## Исправить все SEO-заглушки

Найти и заменить:

https://YOUR-DOMAIN.com

на актуальный адрес сайта.

Исправить:

- canonical

- Open Graph

- Twitter Card

- ссылки в data.js

- privacy.html

- offer.html

- 404.html



## Проверить все внутренние ссылки

Проверить:

- Header

- Footer

- CTA-кнопки

- FAQ

- юридические страницы

Не должно быть битых ссылок.



# ЭТАП 2. Подготовка структуры данных

Все данные должны храниться отдельно от интерфейса.

Создать структуру:

data/

- site.js

- albums.js

- tracks.js

- faq.js

- reviews.js

- pricing.js

- contacts.js

- team.js

- news.js



## Создать новый объект

news

Структура новости:

id

title

excerpt

content

image

date

published



Пока оставить пустым массивом.



# ЭТАП 3. Подготовка слоя данных

Создать новый каталог:

services/



Создать файлы:

albums.service.js

tracks.service.js

orders.service.js

reviews.service.js

faq.service.js

settings.service.js



Пока использовать локальные данные.

Но интерфейс должен обращаться не напрямую к массивам данных, а через сервисы.

Пример:

Плохо:

tracks.forEach(…)

Хорошо:

tracksService.getAll()

Создать каталог:

Создать:

На текущем этапе использовать только local.provider.js

supabase.provider.js оставить как каркас.



# ЭТАП 12. Главный принцип

После завершения работы:

- сайт должен работать как сейчас;

- внешний вид не должен ухудшиться;

- все функции должны работать;

- проект должен быть полностью готов к подключению Supabase;

- подключение Supabase должно потребовать минимум изменений в коде.
